## 一、通用约定

| 项目 | 约定值 |
|------|--------|
| 后端基路径 | `http://localhost:8080/api` |
| 请求格式 | `Content-Type: application/json` |
| Token 传递 | `Authorization: Bearer <token>` |
| 统一响应 | `{ code: number, message: string, data: object }` |
| 错误码 | 200/400/401/403/404/500 |
| 分页请求 | `?page=1&pageSize=10` |
| 分页响应 | `{ records: [], total: 0, page: 1, pageSize: 10 }` |
| 字段命名 | DB `snake_case` → API `camelCase` |
| 密码加密 | BCrypt `$2a$10$` （hutool `cn.hutool.crypto.digest.BCrypt`） |

### 4.2 已解决的历史问题

| 原问题 | 解决方式 |
|--------|---------|
| Token `config.headers.token` 非标准 | 改为 `Authorization: Bearer` |
| 状态码 `30001` 判断过期 | 改为 HTTP 401 拦截 |
| Mock JSON 数据 | 全部替换为 Axios 调用 |
| 注册页无 username | 后端按 `phone` 自动生成 |
| 401 重定向 URL 路径叠加 | `window.location.pathname` 替代 `href` |
| `auth.logout()` 异步竞态 | 移除 `logoutApi` 调用，直接 `clearAuth()` |
| 路由守卫读 Pinia 缓存 | `router.beforeEach` 直接读 `localStorage` |
| 密码 MD5 存储 | 全部替换为 BCrypt（`$2a$10$`） |
| 改密后端不校验 confirmPassword | 已添加后端二次校验 |
