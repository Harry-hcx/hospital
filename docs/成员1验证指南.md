## 任务7：Token 认证与权限拦截

### 测试A：带 Token 获取用户信息

```bash
# 先获取 token
curl -X POST http://localhost:8080/api/auth/login ^
  -H "Content-Type: application/json" ^
  -d "{\"phone\":\"13811112222\",\"password\":\"123456\"}"
```

复制返回的 token 值，然后：

```bash
curl http://localhost:8080/api/auth/me ^
  -H "Authorization: Bearer <粘贴你的token>"
```

**验证**：返回 `{"code":200,"data":{"id":?,"phone":"13811112222","realName":null}}`

### 测试B：不带 Token 访问（拦截验证）

浏览器登录后，打开 `http://localhost:3000`

按 F12 打开开发者工具 → Application → Local Storage → 删除 `token` 和 `userInfo`

然后访问 `http://localhost:3000/change-password`

**验证**：自动跳转到 `http://localhost:3000/login?redirect=%2Fchange-password`

### 测试C：带错误 Token 访问

```bash
curl http://localhost:8080/api/auth/me ^
  -H "Authorization: Bearer invalid-token-12345"
```

**验证**：返回 HTTP 401（或无响应，浏览器控制台看到 401）

## 一键验证脚本（全部 API）

```bash
# 保存为 test.bat 运行
@echo off
echo === 1. 注册 ===
curl -s -X POST http://localhost:8080/api/auth/register -H "Content-Type: application/json" -d "{\"phone\":\"13900001111\",\"password\":\"111111\",\"confirmPassword\":\"111111\"}"
echo.
echo === 2. 登录 ===
curl -s -X POST http://localhost:8080/api/auth/login -H "Content-Type: application/json" -d "{\"phone\":\"13900001111\",\"password\":\"111111\"}" > login_result.txt
type login_result.txt | findstr "token"
echo.
echo === 3. /auth/me (带Token) ===
for /f "tokens=*" %%a in ('powershell -Command "(Get-Content login_result.txt | ConvertFrom-Json).data.token"') do set TOKEN=%%a
curl -s http://localhost:8080/api/auth/me -H "Authorization: Bearer %TOKEN%"
echo.
echo === 4. 改密码 ===
curl -s -X POST http://localhost:8080/api/auth/change-password -H "Content-Type: application/json" -H "Authorization: Bearer %TOKEN%" -d "{\"oldPassword\":\"111111\",\"newPassword\":\"222222\"}"
echo.
echo === 5. /auth/me (无Token) ===
curl -s -w "\nHTTP Status: %%{http_code}\n" http://localhost:8080/api/auth/me
echo.
echo === 全部完成 ===
```
